# NMPA申报材料润色风格指南

## 1. 定位

本指南用于指导 NMPA 申报材料的专业化文字润色。

润色目标是帮助申报方将材料表达得更加正式、规范、清晰和一致，使其更符合注册申报资料的常用语言风格。

本指南不以“审评缺陷筛查”为主要目标，不主动放大问题或增加审评质疑性表述。除非用户明确要求，润色时应以申报方文件的成熟表达、逻辑清晰和专业呈现为核心。

---

## 2. 总体原则

### 2.1 保持原意

润色不得改变原文的核心含义，包括：

- 研究设计
- 研究目的
- 受试者或样本信息
- 给药方案
- 检测方法
- 评价指标
- 统计结果
- 安全性观察结果
- 研究结论

不得添加原文没有的新数据、新判断、新机制解释或新法规依据。

---

### 2.2 表达正式

申报资料应采用正式、客观、稳健的书面表达。

避免：

- 口语化表达
- 宣传性表达
- 过度绝对化表达
- 情绪化评价
- 无依据的价值判断

推荐：

- “结果显示”
- “观察到”
- “研究表明”
- “分析结果提示”
- “总体而言”
- “在本研究条件下”
- “与原文数据一致地表述为……”

---

### 2.3 术语统一

同一文件中，同一概念应尽量保持一致。

重点统一：

- 药品名称
- 药物代号
- 适应症名称
- 研究阶段
- 受试者/患者称谓
- 给药方式
- 疗效终点
- 安全性术语
- 统计分析集
- 研究时间点
- 图表名称
- 缩写和英文全称

---

### 2.4 结论稳健

申报资料应避免过度承诺。润色时应根据原文证据强度选择相应表述。

可以将口语化的积极表达调整为专业表达，但不得将原文结论强度明显放大。

例如：

- “效果很好”可润色为“观察到一定疗效改善”
- “比较安全”可润色为“总体耐受性尚可”或“未观察到新的安全性信号”，前提是原文支持
- “没有问题”可润色为“未发现明显异常”或“未观察到相关异常”，前提是原文支持

---

### 2.5 适度提醒

如原文存在明显术语混用、逻辑不清或表达歧义，可在润色报告中提示“建议确认”。

报告表述宜采用申报方工作语言，例如：

- “建议核对该术语在全文中的一致性”
- “建议确认该处是否指安全性分析集”
- “建议补充对应时间点或表格来源”
- “建议确认该结论是否已有数据支持”

避免在普通润色任务中过度使用：

- “重大缺陷”
- “审评风险较高”
- “不符合申报要求”
- “可能被审评质疑”

除非用户明确要求进行审评风险排查。

---

## 3. 术语规范化

### 3.1 监管与申报术语

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 药监局 | 国家药品监督管理局（NMPA） | 首次出现建议使用全称 |
| 申报文件 | 申报资料 / 注册申报资料 | 根据上下文选择 |
| 交资料 | 提交申报资料 | 正式表达 |
| 补材料 | 补充资料 / 补充提交资料 | 发补回复中常用 |
| 临床批件 | 临床试验批准通知书 / 临床试验默示许可 | 根据具体情形选择 |
| 做临床 | 开展临床试验 | 正式表达 |
| 临床方案 | 临床试验方案（Protocol） | 首次出现可中英文并列 |
| 试验报告 | 临床试验总结报告（CSR） | 根据文件类型选择 |

---

### 3.2 受试者与人群

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 病人 | 受试者 / 患者 | 临床试验中优先用“受试者” |
| 入了多少人 | 纳入多少例受试者 | 正式表达 |
| 筛人 | 筛选受试者 | 正式表达 |
| 用药的人 | 接受给药的受试者 | 明确给药状态 |
| 健康人 | 健康受试者 | 临床研究语境 |
| 掉队 | 脱落 / 退出研究 | 根据实际原因选择 |

---

### 3.3 疗效与有效性

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 有效果 | 观察到疗效改善 / 显示一定有效性 | 根据证据强度选择 |
| 效果很好 | 疗效结果良好 / 显示较好的疗效趋势 | 避免过度宣传 |
| 明显变好 | 较基线改善 / 较对照组改善 | 尽量说明比较对象 |
| 证明有效 | 结果支持其有效性 / 达到预设主要终点 | 需有充分证据 |
| 没有效果 | 未观察到明确疗效改善 | 避免绝对化 |
| 指标下降很多 | 指标较基线下降 / 降低幅度较大 | 保留数据时更好 |

---

### 3.4 安全性

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 副作用 | 不良事件（AE）/ 不良反应（ADR） | 根据因果关系选择 |
| 严重副作用 | 严重不良事件（SAE）/ 严重不良反应 | 根据定义选择 |
| 没有副作用 | 未观察到相关不良事件 / 未发生严重不良事件 | 不建议绝对化 |
| 安全性很好 | 安全性总体可接受 / 耐受性总体良好 | 需原文支持 |
| 毒性不大 | 未观察到明显毒性反应 | 非临床中常用 |
| 没有安全问题 | 未发现新的安全性信号 | 需原文支持 |
| 不良反应很轻 | 不良事件多为轻度或中度 | 保持客观 |

安全性术语应尽量区分：

- AE：不良事件
- ADR：药物不良反应
- SAE：严重不良事件
- TEAE：治疗期间出现的不良事件
- AESI：特别关注不良事件
- DLT：剂量限制性毒性
- MTD：最大耐受剂量

---

### 3.5 药学与质量

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 质量很好 | 质量研究结果支持产品质量可控 | 更适合申报资料 |
| 工艺稳定 | 生产工艺具有一定稳定性 / 工艺稳定性良好 | 需结合数据 |
| 杂质很少 | 杂质水平较低 / 杂质水平符合拟定标准 | 根据原文选择 |
| 批间差不多 | 批间一致性良好 | CMC 常用表达 |
| 放置后没变化 | 稳定性考察期间未见明显变化 | 稳定性表达 |
| 符合要求 | 符合拟定质量标准 / 符合研究要求 | 需明确“要求”来源 |

---

### 3.6 非临床

| 不推荐表达 | 推荐表达 | 说明 |
|---|---|---|
| 动物实验 | 非临床研究 / 动物实验研究 | 视上下文 |
| 没有毒 | 未观察到明显毒性反应 | 避免绝对化 |
| 剂量很高也没事 | 高剂量组未见明显毒性反应 | 正式表达 |
| 安全窗口大 | 暴露裕度较高 / 安全范围较宽 | 需有数据支持 |
| 可以用于人体 | 支持开展后续临床研究 | 避免直接外推 |

---

## 4. 语气正式化

### 4.1 避免口语化

| 原文 | 润色后 |
|---|---|
| 这个试验做了很长时间 | 本试验持续时间较长 |
| 病人吃了药以后 | 受试者给药后 |
| 结果还不错 | 结果显示出较好的趋势 |
| 我们发现 | 研究结果显示 / 观察到 |
| 这个药比较安全 | 本品总体耐受性较好 |
| 数据看起来比较稳定 | 数据显示整体趋势较为稳定 |

---

### 4.2 避免宣传化

| 不推荐表达 | 推荐表达 |
|---|---|
| 突破性疗效 | 显示出潜在疗效优势 |
| 显著优于现有治疗 | 与现有治疗相比具有潜在优势 |
| 安全性极佳 | 安全性总体可接受 |
| 完全解决问题 | 有望改善相关问题 |
| 全球领先 | 具有一定创新性 / 具有进一步研究价值 |
| 疗效非常突出 | 疗效结果较为积极 |

---

### 4.3 避免绝对化

| 不推荐表达 | 推荐表达 |
|---|---|
| 完全没有风险 | 未观察到明显风险 |
| 没有任何不良事件 | 未发生不良事件，或未观察到相关不良事件 |
| 所有患者均获益 | 所有受试者均观察到改善，若原文支持 |
| 一定有效 | 可能具有治疗获益 |
| 绝对安全 | 安全性总体可接受 |
| 无毒性 | 未观察到明显毒性反应 |

---

## 5. 句式优化

### 5.1 长句拆分

如原句超过 60–80 字，且包含多个信息点，应考虑拆分。

原文：

“本研究纳入了多例患者并进行了不同剂量的给药观察，同时也收集了疗效和安全性数据，结果显示药物有一定效果并且整体安全性较好。”

润色后：

“本研究纳入多例受试者，并开展不同剂量水平下的给药观察。研究期间收集了疗效和安全性数据。结果显示，本品在研究人群中观察到一定疗效改善，整体耐受性较好。”

---

### 5.2 明确主语

原文：

“给药后明显改善。”

润色后：

“给药后，受试者的{指标名称}较基线改善。”

如原文未提供指标名称，可润色为：

“给药后，相关评价指标较基线呈改善趋势。”

---

### 5.3 优化逻辑连接

常用连接方式：

- 表示递进：此外、同时、进一步分析显示
- 表示转折：然而、但、需要注意的是
- 表示因果：因此、基于上述结果、结合相关数据
- 表示总结：总体而言、综上、综合现有结果
- 表示条件：在本研究条件下、在当前样本量下、基于现有数据

避免机械堆叠连接词，优先保持自然流畅。

---

## 6. 数字、单位与格式

### 6.1 数字

- 统计数据、样本量、剂量、比例、p 值、置信区间原则上不修改。
- 中文叙述中，具体数值一般使用阿拉伯数字。
- 数字与单位之间建议保留空格，如 `5 mg`、`10 mL`、`2 h`。
- 百分比统一使用 `%`，如 `15.6%`。
- p 值格式可统一为 `p < 0.05` 或 `p = 0.032`。
- 置信区间可按原文格式保持一致，如 `95% CI: x–y`。

### 6.2 缩写

- 专有名词首次出现时，建议写明中文全称、英文全称和缩写。
- 后续可使用缩写。
- 同一缩写在全文中应保持含义一致。
- 不确定缩写含义时，不应自行扩展，应标记为“建议确认”。

### 6.3 药品名称

- 优先使用通用名。
- 代号、商品名、批号、规格等不得随意修改。
- 如同一药品存在多个名称，应建议全文统一。
- 首次出现时可采用“通用名（商品名，若适用）”或“药物代号（通用名，若适用）”。

---

## 7. 模块化润色重点

### 7.1 临床试验方案

重点优化：

- 研究目的表述
- 入排标准表述
- 给药方案表述
- 终点定义
- 安全性观察
- 统计分析描述

推荐表达：

“本研究旨在评价{药品名称}在{目标人群}中的安全性、耐受性、药代动力学特征及初步疗效。”

---

### 7.2 临床研究总结

重点优化：

- 研究设计
- 受试者构成
- 暴露情况
- 疗效结果
- 安全性结果
- 研究结论

推荐表达：

“在本研究条件下，{药品名称}在{目标人群}中显示出可接受的安全性和耐受性，并观察到一定疗效改善趋势。”

如原文已经有确证性统计结果，可根据原文增强为：

“研究达到预设主要终点，结果支持{药品名称}用于{适应症}的有效性。”

---

### 7.3 发补回复

发补回复应简洁、直接、可定位。

推荐结构：

1. 先回应问题。
2. 再说明补充或修订内容。
3. 最后给出资料位置。

示例：

“申请人已根据审评意见补充{研究/说明/数据}。相关内容已更新至申报资料第{章节号}章，并在附件{编号}中提供支持性资料。”

---

### 7.4 药学资料

重点优化：

- 处方组成
- 生产工艺
- 质量标准
- 分析方法
- 杂质控制
- 稳定性
- 批间一致性

推荐表达：

“研究结果显示，拟定生产工艺具有较好的可重复性，关键质量属性在研究批次间保持相对一致。”

---

### 7.5 非临床资料

重点优化：

- 模型和种属
- 剂量设置
- 给药途径
- 暴露量
- 毒性观察
- 安全性结论

推荐表达：

“在本研究条件下，未观察到与受试物相关的明显毒性反应，相关结果支持后续临床研究的开展。”

---

### 7.6 安全性资料

重点优化：

- AE/SAE/ADR 术语
- 发生率描述
- 严重程度
- 与药物相关性
- 转归
- 是否导致停药或退出
- 是否存在新的安全性信号

推荐表达：

“研究期间报告的不良事件多为轻度或中度。未观察到与研究药物相关的严重不良事件，整体安全性特征与预期一致。”

前提：原文应支持上述表述。

---

## 8. 常见润色场景

### 场景 1：研究目的

原文：

“本研究想看看这个药对病人有没有效果。”

润色后：

“本研究旨在评价{药品名称}治疗{适应症}的有效性和安全性。”

如为早期临床，可润色为：

“本研究旨在评价{药品名称}在{目标人群}中的安全性、耐受性、药代动力学特征及初步疗效。”

---

### 场景 2：受试者描述

原文：

“一共入了100个病人，男女都有。”

润色后：

“本研究共纳入100例受试者，包括男性和女性受试者。”

---

### 场景 3：疗效结果

原文：

“吃了药以后，大部分病人的症状都好多了。”

润色后：

“给药后，多数受试者的相关症状较基线有所改善。”

如原文有具体指标，应保留指标：

“给药后，多数受试者的{指标名称}较基线改善。”

---

### 场景 4：安全性

原文：

“副作用不大，就是有点头疼、恶心。”

润色后：

“安全性方面，报告的不良事件主要包括头痛和恶心，多数为轻度或中度。”

如原文明确无 SAE，可补充：

“研究期间未报告严重不良事件。”

---

### 场景 5：统计结果

原文：

“治疗组比对照组好一些，但没有显著。”

润色后：

“与对照组相比，治疗组在{指标名称}方面呈改善趋势，但组间差异未达到统计学显著性。”

---

### 场景 6：质量研究

原文：

“几批样品检测结果差不多，说明工艺还可以。”

润色后：

“多批样品的检测结果较为一致，提示拟定生产工艺具有一定可重复性。”

---

## 9. 保持不变的内容

以下内容原则上不应修改：

- 统计数据
- 样本量
- 剂量和单位
- 给药频次
- 时间点
- p 值
- 置信区间
- 终点定义
- 分析集名称
- 药品名称和代号
- 批号
- 法规条款编号
- 参考文献
- 章节编号
- 图表编号
- 表格中的原始数据
- 公式
- URL
- 交叉引用字段

如发现明显笔误，可在报告中提示，由用户确认是否修改。

---

## 10. 润色报告建议

润色报告采用申报方辅助工作口吻，重点说明“改了什么”和“还有什么建议确认”。

建议结构：

### 10.1 润色概况

- 本次润色范围
- 处理段落数量
- 跳过内容说明

### 10.2 主要修改类型

- 术语统一
- 语气正式化
- 句式优化
- 逻辑衔接优化
- 安全性表述规范化
- 数字和单位格式统一

### 10.3 典型修改示例

| 原文 | 润色后 | 修改说明 |
|---|---|---|
| 口语化表达 | 正式申报表达 | 提升正式性 |
| 术语不统一 | 统一术语 | 保持全文一致 |
| 长句 | 拆分短句 | 提升可读性 |

### 10.4 建议确认事项

建议使用温和表述：

- “建议确认该处是否指……”
- “建议核对全文是否统一使用该术语。”
- “建议确认该结论是否需要补充对应数据来源。”
- “建议确认该时间点与表格数据是否一致。”

---

## 11. 推荐整体风格

润色后的申报资料应呈现以下风格：

- 正式但不僵硬
- 专业但不晦涩
- 稳健但不消极
- 清晰但不删减关键信息
- 客观但保留申报方合理表达
- 规范但不过度审评化

最终目标是帮助申报方形成一份语言成熟、逻辑清楚、术语统一、表达稳健的申报材料。
